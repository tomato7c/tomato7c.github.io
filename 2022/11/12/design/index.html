技术方案设计中要考虑的几个元素
2022-11-12 19:11:41 #design

1. Background / Introduction

为什么要做这件事

2. Requirements vs non-functional requirements

明确好需求和设计的目的

3.Proposed solution and alternatives

先说方案，然后表名自己做过 research，但别的选择没有我的好，借助架构图方便大家理解。Proposed solution 需要非常详细，需要包括数据协议与接口协议，相应的改动点。alternative 可以相对简单，但要明确优劣。
如果系统比较大，拆分成不同 components, 每个 components 包含 proposed soltion 与 alternatives。

4. Test / Monitoring / Metric

怎么做测试，监控系统，哪些 metric 可以衡量项目的成功

5. Work items

预估每部分需要多长时间, 多少 engineer week。
ps: 知乎上搜索来的名词，竟然以 week 作为单位，羡慕

6. Questions / Meeting notes

design review 时候记录一些别人的问题

7. Reference

相关的doc, 比如需要碰到的系统等等
